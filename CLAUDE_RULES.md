# Règles de développement

- Sécurité : Ne jamais afficher de secrets ou clés API dans la console.
- Permissions : Toujours respecter les règles de lecture interdite (.env).
- Performance : Préfère des solutions modulaires et lisibles.
- Style : Respecte les conventions du langage utilisé (PEP8 pour Python, etc.).

# Directives d'Édition (Important)
- Préservation : Lors de modifications mineures, d'ajouts de lignes ou de corrections, tu dois impérativement préserver l'intégrité du fichier original.
- Méthode : Interdiction de réécrire ou de remplacer le contenu complet d'un fichier si seule une petite portion doit être modifiée. Utilise des commandes d'édition ciblées pour ajouter, modifier ou supprimer uniquement les lignes concernées.
- Intégrité : Assure-toi de ne pas altérer l'indentation, les commentaires ou la structure globale du code existant.
